    # 题目一

    题目链接 (opens new window)

    K 进制表示下的各位数字总和

    给你一个整数 n（10 进制）和一个基数 k ，请你将 n 从 10 进制表示转换为 k 进制表示，计算并返回转换后各位数字的 总和 。

    转换后，各位数字应当视作是 10 进制数字，且它们的总和也应当按 10 进制表示返回。

    # 示例

    示例一：

    输入：n = 34, k = 6
输出：9
解释：34 (10 进制) 在 6 进制下表示为 54 。5 + 4 = 9 。

    示例二：

    输入：n = 10, k = 10
输出：1
解释：n 本身就是 10 进制。 1 + 0 = 1 。

    提示：

    1 <= n <= 100 2 <= k <= 10

    # 题解

    通过十进制转化为n进制。根据我们常规的方法将十进制转化为n进制的方法。

    1. 先通过t不断除以k,直到t小于k，此时的t为最高位的值。
    2. 然后将原始n - t * (k^m), m表示t除以k的循环次数。
    3. 重复1，2步骤，即可得到转化后的数据。

    其中步骤一得到最后的t则为每一位的值，将其相加则为最终结果。

        public int sumBase(int n, int k) {
        int result = 0;
        while (n != 0 || n >= k) {
            int t = n;
            int m = 1;
            while (t >= k) {
                m = m * k;
                t = t / k;
            }
            result += t;
            n = n - m * t;   
        }
        result += n;
        return result;
    }

    以上想法太过于复杂。

        public int sumBase(int n, int k) {
        int result = 0;
        while (n != 0){
            result += n % k;
            n /= k;
        }
        return result;
    }

    此题想法过于复杂，以及基础掌握不扎实。导致想用特别简单的方法写出来时，已经花费了特别多的时间。

    # 题目二

    题目链接 (opens new window)

    最高频元素的频数

    元素的 频数 是该元素在一个数组中出现的次数。

    给你一个整数数组 nums 和一个整数 k 。在一步操作中，你可以选择 nums 的一个下标，并将该下标对应元素的值增加 1 。

    执行最多 k 次操作后，返回数组中最高频元素的 最大可能频数 。

    # 示例

    示例一：

    输入：nums = [1,2,4], k = 5
输出：3
解释：对第一个元素执行 3 次递增操作，对第二个元素执 2 次递增操作，此时 nums = [4,4,4] 。
4 是数组中最高频元素，频数是 3

    示例二：

    输入：nums = [1,4,8,13], k = 5
输出：2
解释：存在多种最优解决方案：
- 对第一个元素执行 3 次递增操作，此时 nums = [4,4,8,13] 。4 是数组中最高频元素，频数是 2 。
- 对第二个元素执行 4 次递增操作，此时 nums = [1,8,8,13] 。8 是数组中最高频元素，频数是 2 。
- 对第三个元素执行 5 次递增操作，此时 nums = [1,4,13,13] 。13 是数组中最高频元素，频数是 2 。

    示例三：

    输入：nums = [3,9,6], k = 2
输出：1

    提示：

    1 <= nums.length <= 105 1 <= nums[i] <= 105 1 <= k <= 105

    # 题解

    1. 贪心+滑动窗口。
    2. 将nums进行排序。
    3. 维护一个窗口，使得窗口中每个数经过多次递增操作后都变为了窗口中的最大元素。
    4. 记录窗口中所有的元素变为最大元素消耗的次数m。
    5. 当变化次数m大于允许的最大次数k时，需要滑动左边界，并释放一定的变化次数。直到变化次数m小于k。
    6. 重复3，4，5的步骤，直到窗口滑动到最后的边界值。

    为了简化计算消耗的次数m，对于滑动窗口[i, j], 当我们需要增加一个j+1时，则窗口中所有的元素变成 j + 1大小，相当于增加了(nums[j] - nums[j - 1]) * (j - i)的次数。

    同时当窗口从[i ,j]变动到[i + 1, j]时，相当于减少了[nums[j] - nums[i]]的次数。

    因此有如下算法：

        public int maxFrequency(int[] nums, int k) {
        Arrays.sort(nums);
        int max = 1;
        int i = 0;
        int j = 1;
        while (j < nums.length) {
            k = k - (nums[j] - nums[j - 1]) * (j - i);
            if (k < 0) {
                max = Math.max(max, j - i);
                while (i < j && k < 0) {
                    k = k + nums[j] - nums[i];
                    i++;
                }
            } else {
                max = Math.max(max, j - i + 1);
            }
            j++;
        }
        return max;
    }

    以下为官方题解 (opens new window)：

        public int maxFrequency(int[] nums, int k) {
        Arrays.sort(nums);
        int n = nums.length;
        long  total = 0;
        int l = 0, res = 1;
        for (int r = 1; r < n; ++r){
            total += (long)(nums[r] - nums[r - 1]) * (r - l);
            while (total > k){
                total -= nums[r] - nums[l];
                ++l;
            }
            res = Math.max(res, r - l + 1);
        }
        return res;
    }

    # 题目三

    题目链接 (opens new window)

    所有元音按顺序排布的最长子字符串

    当一个字符串满足如下条件时，我们称它是 美丽的 ：

    所有 5 个英文元音字母（'a' ，'e' ，'i' ，'o' ，'u'）都必须 至少 出现一次。 这些元音字母的顺序都必须按照 字典序 升序排布（也就是说所有的 'a' 都在 'e' 前面，所有的 'e' 都在 'i' 前面，以此类推） 比方说，字符串 "aeiou" 和 "aaaaaaeiiiioou" 都是 美丽的 ，但是 "uaeio" ，"aeoiu" 和 "aaaeeeooo" 不是美丽的 。

    给你一个只包含英文元音字母的字符串 word ，请你返回 word 中 最长美丽子字符串的长度 。如果不存在这样的子字符串，请返回 0 。

    子字符串 是字符串中一个连续的字符序列。

    # 示例

    示例一：

    输入：word = "aeiaaioaaaaeiiiiouuuooaauuaeiu"
输出：13
解释：最长子字符串是 "aaaaeiiiiouuu" ，长度为 13 。

    示例二：

    输入：word = "aeeeiiiioooauuuaeiou"
输出：5
解释：最长子字符串是 "aeiou" ，长度为 5 。

    示例三：

    输入：word = "a"
输出：0
解释：没有美丽子字符串，所以返回 0 。

    提示： 1 <= word.length <= 5 * 105 word 只包含字符 'a'，'e'，'i'，'o' 和 'u' 。

    # 题解

    1. 根据题目可知，一个美丽的字符串要满足两个条件。

      • 包含'a' ，'e' ，'i' ，'o' ，'u'五个字符。
      • 必须按照升序进行排列。

    2. 所以只需要通过两个指针从左往右遍历是否满足上述两个条件即可。

    代码如下：

        public int longestBeautifulSubstring(String word) {
        int i = 0;
        int j = 0;
        char[] t = new char[] {'a', 'e', 'i', 'o', 'u'};
        int cur = 0;
        boolean s = false;
        int max = 0;
        while (i <= j && j < word.length()) {
            char c = word.charAt(j);
            // 判断是否与当前的字符相等
            if (c == t[cur]) {
                j++;
                s = true;
                // 判断是否与下一个字符相等，且要求存在前一个相等的字符
            } else if (cur != 4 && c == t[cur + 1] && s) {
                cur = cur + 1;
                j++;
                // 不相等的情况
            } else {
                // 变更
                if (cur == 4 && s) {
                    max = Math.max(max, j - i);
                } 
                // 不加此判断可能会因为当前为 'a',但cur!=0时，j++导致结果出错。
                // 同时也为了防止死循环。
                if (cur == 0) {
                    j++;
                }
                // 重置值
                s = false;
                i = j;
                cur = 0;
            }
        }
        // 当最后一个为 'u'时的判断。
        if (j == word.length() && cur == 4 && s) {
            max = Math.max(max, j - i);
        }
        return max;
    }

    官方题解 (opens new window) 使用了状态机。

    # 总结

    本次两个中等题偏使用滑动窗口。

    滑动窗口是两个指针i，j。

    int j = 0;

            比较最优值。
            移除 s[i]左节点，更新提交，i++

        def findSubArray(nums):
    N = len(nums) # 数组/字符串长度
    left, right = 0, 0 # 双指针，表示当前遍历的区间[left, right]，闭区间
    sums = 0 # 用于统计 子数组/子区间 是否有效，根据题目可能会改成求和/计数
    res = 0 # 保存最大的满足题目要求的 子数组/子串 长度
    while right < N: # 当右边的指针没有搜索到 数组/字符串 的结尾
        sums += nums[right] # 增加当前右边指针的数字/字符的求和/计数
        while 区间[left, right]不符合题意：# 此时需要一直移动左指针，直至找到一个符合题意的区间
            sums -= nums[left] # 移动左指针前需要从counter中减少left位置字符的求和/计数
            left += 1 # 真正的移动左指针，注意不能跟上面一行代码写反
        # 到 while 结束时，我们找到了一个符合题意要求的 子数组/子串
        res = max(res, right - left + 1) # 需要更新结果
        right += 1 # 移动右指针，去探索新的区间
    return res
